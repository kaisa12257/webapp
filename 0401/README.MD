컴포넌트
1. 리액트의 심장: 컴포넌트(Component)
정의: UI를 이루는 가장 작은 독립된 부품이자 모듈입니다.

역할: 입력을 받아서 화면에 보여줄 요소(Element)를 출력합니다. 자바스크립트의 함수와 비슷하게 작동하여 HTML 엘리먼트를 반환합니다.

2. 조립식 구조 (레고 & 퍼즐)
레고 블록: 마치 레고를 조립하듯 작은 컴포넌트들을 끼워 맞춰 복잡한 웹 페이지를 완성합니다.

재사용성: 한 번 잘 만들어둔 컴포넌트는 여기저기서 다시 쓸 수 있어 매우 효율적입니다.

3. 특징과 장점
일관성: 어떤 데이터 집합을 넣더라도 같은 컴포넌트를 사용하면 항상 동일한 구조(DOM)를 만들어냅니다.

구조화: 웹 페이지 전체를 하나의 큰 그림으로 보고, 이를 컴포넌트라는 조각으로 나누어 설계하므로 관리가 쉽습니다.

1. 리액트 컴포넌트의 메커니즘 (입력 → 처리 → 출력)
리액트 컴포넌트는 마치 붕어빵 틀이나 함수처럼 작동해요.

입력 (props): 컴포넌트에 전달되는 속성(Property) 값이에요. 붕어빵으로 치면 '팥'을 넣을지 '슈크림'을 넣을지 결정하는 재료라고 보시면 돼요.

출력 (React Element): 입력받은 props를 바탕으로 만들어진 결과물이에요. 화면에 실제로 그려질 정보를 담고 있는 자바스크립트 객체 형태죠.

2. 'props'라는 이름의 의미
어원: property(속성, 특성)의 줄임말인 prop에서 왔어요.

복수형: 보통 속성이 여러 개일 수 있기 때문에 뒤에 s를 붙여 **props**라고 불러요.

역할: 컴포넌트의 성격을 규정짓는 데이터예요. 예를 들어 '버튼' 컴포넌트에 color="blue"라는 props를 주면 파란 버튼이 만들어지는 식이죠.

3. 핵심 포인트 요약
빌딩 블록: React Element는 리액트 앱을 구성하는 가장 작은 단위(벽돌)예요.

자동화: 관리자님이 원하는 대로 props를 넣기만 하면, 리액트 컴포넌트가 그 속성에 맞춰 화면에 나타날 Element를 알아서 생성해서 리턴해 줍다.

1. 함수 컴포넌트(Function Component)란?
말 그대로 자바스크립트의 **'함수'**를 사용해서 만든 컴포넌트야.

이미지 속 예시처럼 function Welcome(props) { ... }와 같은 형태를 가져.

입력: 하나의 props 객체를 인자로 받아.

출력: 인사말이 담긴 <h1> 태그 같은 React Element를 리턴해.

2. 핵심 개념: 순수 함수(Pure Function)
리액트 컴포넌트는 '순수 함수' 같은 역할을 해야 해.

순수 함수란? 똑같은 입력(props)이 들어오면 항상 똑같은 결과물(화면)을 내보내야 한다는 뜻이야. (중간에 딴짓을 하거나 입력값을 직접 바꾸면 안 돼!)

3. 함수 컴포넌트의 장점
간결함: 코드가 아주 짧고 직관적이야. (클래스 컴포넌트보다 훨씬 간단해!)

가독성: 딱 보면 "아, 이 데이터가 들어와서 이 화면이 나오는구나"라고 바로 이해하기 쉽다.

컴포넌트는 항상 대문자로 시작해야된다.
소문자라면 DCM으로인식한다.
